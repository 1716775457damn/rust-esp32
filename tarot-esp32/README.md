# Tarot ESP32-C3: 高清数字塔罗抽卡机

这是一个基于 **ESP32-C3** 开发的高性能数字塔罗抽卡项目。项目采用 **Rust + C++ 混合架构**，成功在内存仅有 400KB 的 C3 芯片上驱动了 **360x360 高分辨率圆屏**。

## ✨ 特性

- **高清显示**: 支持 360x360 高分辨率 ST77916 圆屏，色彩饱满，字迹清晰。
- **混合架构**: 
  - **Rust**: 负责业务逻辑、Wi-Fi AP Server、HTTP 协议栈以及 JPEG 动态解码。
  - **C++**: 负责底层硬件驱动（SPI）、LVGL UI 框架以及图像流渲染。
- **内存优化 (Streaming Render)**: 独创的“分块流渲染”逻辑，每次仅占用 14KB 内存即可渲染 253KB 的全屏图像。
- **精美 UI**: 提供移动端友好的抽卡网页，支持中文卡片名显示。

## 🛠️ 硬件需求

- **主控**: ESP32-C3 (建议 16MB Flash 版本)
- **屏幕**: 1.8" 圆形 LCD/AMOLED (驱动芯片 ST77916)
- **引脚接线**:
  - **SCLK**: GPIO 1
  - **MOSI**: GPIO 2
  - **DC**: GPIO 0
  - **CS**: GPIO 21
  - **BL (背光)**: GPIO 20
  - **RST (复位)**: GPIO 3 (部分硬件可能通过 I2C 扩展器复位)

## 🚀 快速开始

### 1. 准备资源
将你的塔罗牌图片（JPG 格式）放入目录并运行脚本：
```powershell
python prepare_assets.py
```
该脚本会自动将图片缩放为 360x360 并存入 `spiffs_data`。

### 2. 编译并烧录
项目使用 ESP-IDF v5.3.1 开发：
```powershell
idf.py add-dependency "espressif/esp_lcd_gc9a01"
idf.py build flash monitor
```

### 3. 使用方法
1. 手机连接 Wi-Fi 热点：`TarotCard`。
2. 浏览器访问：`192.168.71.1`。
3. 点击“Draw a Card”按钮，屏幕将即时刷新卡片。

## 💎 技术核心 (The "Hard-Won" Knowledge)

### 1. 攻克 C3 内存瓶颈
ESP32-C3 的连续内存非常有限。一张 360x360 RGB565 的图像需要 259,200 字节，直接加载会导致 OOM。
**解决方案**：在渲染时，我们不一次性加载全图，而是分 18 次读取，每次只从文件系统提取 20 行像素并立即推送到 SPI 总线。

### 2. ST77916 驱动黑科技
本项目针对 ST77916 的特殊协议做了以下处理：
- **借壳驱动**：使用 `gc9a01` 的驱动骨架配合 ST77916 的特定指令集，绕过了官方库的某些时序问题。
- **专家序列**：移植了包含 187 条厂家寄存器指令的深度初始化序列。
- **时钟极性**：稳定运行在 SPI Mode 0, 20MHz。
