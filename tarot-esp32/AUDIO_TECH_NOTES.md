# 🎴 塔罗机音频硬件调试技术手册 (ES8311 + ESP32-C3)

本手册记录了针对“立创 C3 开发板”架构在开发过程中遇到的核心音频问题及其最终解决方案。

## 1. 核心问题：持续的电流声 (Static/Hiss Noise)
**现象**：播放时只有沙沙的噪音，或者在静默期有微小的电流声。

### 解决方案：时钟位对齐 (PLL Alignment)
*   **无 MCLK 模式**：由于该硬件未连接 MCLK 引脚，ES8311 必须从 I2S 的 BCLK 信号中提取采样时钟。
*   **PLL 倍频锁定**：必须手动配置 `REG02`。
    *   **关键寄存器**：`es8311_write_reg(0x02, 0x18)`。
    *   **原理**：`0x18` 开启了内部时钟的 x8 倍频模式。没有这个设置，DAC 的内部时钟频率不足，无法同步 BCLK，导致解码破碎。
*   **I2S 协议对齐**：在 ESP32-C3 硬件上，即使使用 Philips 标准，也必须显式设置 `left_align = true`，否则 16bit 数据在 32bit 槽位中会出现偏移。

## 2. 核心问题：播放瞬间重启 (BOD Reset)
**现象**：抽卡播放音效的同时，系统触发 Brownout Detector 重启。

### 解决方案：电力避让 (Power Management)
*   **背光冲突**：屏幕背光与音频功放在峰值功率时会争抢 USB 供电。
*   **代码逻辑**：在 `play_wav` 函数中，播放前通过 `gpio_set_level(GPIO_NUM_7, 0)` 关闭/调暗背光，播放结束后恢复。

## 3. 标准寄存器配置表 (16kHz, Stereo-Clone)
| 寄存器 | 路径 | 设定值 | 作用描述 |
| :--- | :--- | :--- | :--- |
| `0x01` | Clock | `0xBF` | 开启全域时钟电源 |
| `0x02` | PLL | `0x18` | **锁定 BCLK 同步 (核心关键)** |
| `0x09` | Format | `0x0C` | I2S Philips 16-bit 格式 |
| `0x32` | Volume | `0x90` | 数字音量 (0x90 约为 55%) |
| `0x14` | Gain | `0x1A` | 模拟前端增益 |
| `0x44` | I2C | `0x08` | 增强 I2C 总线抗干扰 (需双写) |

## 4. 音频素材标准
*   **格式**：WAV (Unsigned 16-bit PCM)
*   **采样率**：16,000 Hz (经过实测，该频率在无 MCLK 环境下最为稳健)
*   **通道**：单声道 (工程代码内部会自动将其克隆为双声道信号发送给 I2S)
